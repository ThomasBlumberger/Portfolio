Homework 1: Tasks for tb278

Your personal task is to develop a system of hotels and the rooms inside these hotels.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Room. With the following private fields:

  1. String name
  2. String description
  3. String category
  4. int guests
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Hotel. With the following private fields:

  1. int id
  2. String name
  3. String description
  4. List<Room> rooms
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Hotel> hotels in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one hotel with two rooms. [4 marks]

Create controller co2103.hw1.controller.HotelController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/hotels" should show view hotels/list. [2 marks]
    • Add the list of all hotels from co2103.hw1.Hw1Application to the model as attribute hotels. [2 marks]
    • Create the corresponding JSP to list the name and description of each hotel followed by a link to URL "/rooms?hotel={id of hotel}", e.g., "/rooms?hotel=3". [4 marks]
  2. Requests for URL "/newHotel" should show view hotels/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="hotel" and action="/addHotel" and inputs for all fields of class Hotel except for rooms. [5 marks]
  3. POST requests for URL "/addHotel" should add a new hotel to the list of all hotels in co2103.hw1.Hw1Application and redirect to /hotels. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.RoomController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/rooms" should show view rooms/list. [2 marks]
    • Based on request parameter hotel determine the corresponding hotel from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all rooms of that hotel to the model as attribute rooms. [2 marks]
    • Create the corresponding JSP for the view to list the name, description, and category of each room. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newRoom?hotel={id of hotel whose rooms are listed}", e.g., "/newRoom?hotel=3". [4 marks]
  2. Requests for URL "/newRoom" should show view rooms/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="room" and action="/addRoom" and inputs for all fields of class Room. [6 marks]
    • Also add the request parameter hotel with the ID of the current hotel. [4 marks]
  3. POST requests for URL "/addRoom" should add a new room to the list of rooms of the hotel identified by request parameter hotel and then redirect to /hotels. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Hotel and make it available in controller HotelController. [4 marks]

When adding a new hotel validate that

  1. The value of field id is unique within the list Hw1Application.hotels. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field description is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Room and make it available in controller RoomController. [4 marks]

When adding a new room validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field description is not empty. [2 marks]
  3. The value of field category is either "suite", "penthouse", or "simple". [3 marks]
  4. The value of field guests is between (including) 1 and 12. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
